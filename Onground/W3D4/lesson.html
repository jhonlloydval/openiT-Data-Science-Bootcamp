W3D4 Lesson

Welcome to the W3D4 lesson. This page explains the whole folder in a simple and beginner-friendly way. If you are new to transformers, Streamlit, and agent-style tool calls, this lesson walks you through the ideas step by step.

What is W3D4?

W3D4 is a small Streamlit app that teaches you how to build a chat interface and how to connect it to a local model service (like Ollama). The app shows different examples of:

  • basic chat requests
  • chat user interfaces
  • conversation history
  • streaming responses
  • tool calling / function calls
Big idea: W3D4 is not only about the model answer. It's also about how the app sends requests, how it keeps chat history, and how it lets the model use extra tools.

How the app works

The main file is app.py. It is the starting page for the Streamlit app. The extra examples are in W3D4/pages/. Each file is its own page in Streamlit.

Every example uses a similar pattern:

  • create a prompt input area
  • build a JSON payload for the model
  • send the payload to the local server
  • show the response on the page

Run this app

  1. Install the needed packages: pip install streamlit requests
  2. Start the local model service (Ollama) so it listens at http://localhost:11434
  3. Run: streamlit run app.py
Note: The example code expects the model server at localhost:11434. If that server is not running, the pages will not return answers.

Page summary

1. Basic

This page shows the first, simplest chat example.

payload = {
  "model": "llama3.2",
  "messages": messages,
  "stream": False
}
response = requests.post("http://localhost:11434/api/chat", json=payload)
reply = response.json()
st.write(reply)

It sends a request and prints the full JSON response. The activity asks you to find the assistant text inside that response.

2. UI

This page uses Streamlit chat UI components to show how a chat interface can look.

with st.chat_message("system"): st.markdown("You are a helpful assistant.")
with st.chat_message("user"): st.markdown("This is a user message.")
with st.chat_message("assistant"): st.markdown("This is an assistant message.")

It is just a display example, but it helps you see how chat bubbles are built.

3. Conversation

This page saves the conversation in st.session_state. That means the app remembers what you and the assistant said earlier.

if "messages" not in st.session_state:
    st.session_state.messages = [
        {"role": "system", "content": "You are a helpful assistant."}
    ]

for msg in st.session_state.messages:
    with st.chat_message(msg["role"]): st.markdown(msg["content"])

Then it adds each new user message and assistant reply to the history so the chat keeps going.

4. Streaming

This page shows how to get a response as it arrives, chunk by chunk.

payload = {
  "model": "llama3.2",
  "messages": st.session_state.messages,
  "stream": True
}
with requests.post("http://localhost:11434/api/chat", json=payload, stream=True) as response:
    for line in response.iter_lines():
        if line:
            chunk = json.loads(line)
            st.write(chunk)

The app prints each streamed piece so you can see the model answer grow over time.

5. Tool Calling

This page teaches a new idea: the model can choose to call a tool instead of answering directly.

tools = [
  {
    "type": "function",
    "function": {
      "name": "get_weather",
      "description": "Get the current weather for a given city",
      "parameters": { ... }
    }
  }
]

The model sees these tools and can decide to use one. The response will include a tool call if it wants that help.

Key beginner concepts

Messages and roles

The request body often uses a list called messages. Each message has a role:

  • system: sets the assistant's behavior
  • user: the question or prompt you type
  • assistant: the model's reply

Why the system prompt matters

The system prompt is how you tell the model what kind of assistant to be. For example, ask it to act like a pirate or a cat:

{"role": "system", "content": "You are a helpful assistant."}

What is a model payload?

The payload is the data sent to the model server. Common fields are:

  • model: which model to use
  • messages: chat history
  • stream: true or false
  • tools: optional tool definitions for the model

Tool calling in simple words

Normally the model replies with text. With tool calling, the model can instead say:

"Please run this tool with these inputs."

Then the app runs the tool and returns the tool result to the user.

Why this lesson is useful

W3D4 teaches three main beginner skills:

  1. how to build chat UI pages with Streamlit
  2. how to send model requests and read responses
  3. how to add tools so your assistant can do things beyond just talking
Try this first: run the Basic page, ask a simple question, then open the JSON response and find only the assistant answer inside it.

Quick checklist for a beginner

  • Open W3D4/app.py and see the home page text.
  • Open W3D4/pages/1 1 Basic.py and study the request code.
  • Open W3D4/pages/2 2.2 Conversation.py to understand chat history.
  • Open W3D4/pages/4 4.4 Tool Function Call.py to learn how a tool is defined and used.
  • Run the app with streamlit run app.py and click the pages.

Final advice

The easiest way to learn is to change one thing at a time. For example:

  • change the system prompt
  • change the model name to llama3.2
  • print only the assistant text instead of the full JSON
  • make the tool return a different string

When you are ready, try combining the chat history and tool calls in the same page.
